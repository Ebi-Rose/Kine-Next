Progress — A + B Combined

Hero card on top (A's anchor), then a dashboard grid (B's scannability), then tabs to switch lens. The page is one screen of glance + a few rows of taps.

The combine

  • From A: a single hero card that answers the headline question, plus tabs that segment Strength / Body / History so the page never grows past one screen of scroll.
  • From B: the dashboard grid of small purposeful cards underneath the hero — each tappable into a focused detail screen.
  • The hybrid: hero is a verdict (Story Feed influence: leads with "Stronger by 8%"), grid handles breadth without bloating the hero, tabs handle depth without stacking three full sections.

What it solves

  • A alone risked hiding too much behind tabs — the Body tab might never get discovered.
  • B alone risked card-soup with no clear visual anchor — six equal tiles compete.
  • Combined: one hero leads the eye, the grid offers parallel entry points, the tabs let us add Body / History without redesigning.

Body tab. Hero leads with why photos matter ("strongest record of change"), reassures it's private and never required. Empty state is a clear invitation, not a demand. Measurements aren't tracked yet — surfaced as an opt-in toggle, not a default card.

Why this might be the answer

  • One screen above the fold. Hero + tabs + 4-tile grid all fit before the user scrolls. They get the headline immediately.
  • Tabs don't hide critical content because the hero card carries the most important fact for whichever tab you're on.
  • The grid scales. If we add a card later (mobility, rehab, cycle lens), it slots into the grid without redesigning.
  • The personalization engine drops in cleanly. Hero variant + grid card list + tab visibility all come from computeProgressLayout(). Same structure handles all 8 archetypes.

Trade-offs

  • More structure than C's story feed — slightly less voice. Fix by writing every eyebrow and stat as a verdict.
  • Body tab discoverability is still a question. Mitigation: when there's something noteworthy in Body (new photo, new measurement), badge the tab.
