Design decisions

  • Warm cream, not white. #f6f1ea bg keeps Kine's warmth and reads feminine; pure #fff would feel clinical and break brand.
  • Accent darkened for contrast. The rose #c49098 fails AA on cream, so light-mode accent is #8e5560 — same hue family, just weighted for legibility. Buttons keep filled rose; links/active states use it too.
  • Hero texture softened. The hero-bg-opt.jpg overlay drops from 0.06 → 0.04 opacity — on light backgrounds it has more visible impact.
  • Category accents (sage/gold/blue) darken in parallel so they stay distinguishable against cream and maintain semantic meaning across both themes.
  • Borders visible. #dcd2c4 gives card edges definition without harsh lines — matches the soft feel of the dark theme's muted borders.
  • Filled accent buttons invert. Dark mode: rose bg + dark text. Light mode: dark-rose bg + cream text. Maintains the same visual weight.

Implementation notes (not in scope yet)

  • Add a .light class on <html> that overrides the same CSS custom properties defined in globals.css.
  • Toggle lives in Profile; default to system preference via prefers-color-scheme on first load, persist choice in localStorage.
  • Every component reads from --color-* tokens already — no per-component refactoring needed beyond any hardcoded hex values (audit required).
  • Sentry/error screens and onboarding backgrounds that use custom gradients will need parallel light values.
